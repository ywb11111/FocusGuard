# FocusGuard 项目背景与发现

## 项目简介

FocusGuard 是一个 Kotlin + Jetpack Compose Android App，用于专注与环境监测。它应记录专注时长、环境噪声、光照、手机移动、应用使用/分心情况，并生成报告。

## 目标技术深度

- 使用 SensorManager 获取光照和加速度计数据。
- 使用 AudioRecord 采集相对环境噪声。
- 使用 Foreground Service 和通知维持后台监测。
- 使用 Room 保存专注记录和采样数据。
- 使用 DataStore 保存用户设置。
- 使用 WorkManager 生成每日总结。
- 使用 StateFlow / SharedFlow 组织状态和事件。
- 使用 Repository 隔离 UI 与数据源。
- 处理 Compose Canvas 或图表高频数据导致的重组和性能问题。

## 当前仓库状态

- 项目名：FocusGuard。
- 包名：`com.ywb.focusguard`。
- 已有基础 Compose Android 工程。
- 已搭建分层包结构：`data`、`domain`、`service`、`ui`、`di`。
- 已接入 Navigation Compose、Hilt、Room、DataStore、WorkManager、KSP 等基础依赖。
- 当前 Repository 仍包含演示数据，真实 Room 读写尚未完成。

## 用户背景

- 用户是软件工程专业大三学生，方向为 Android。
- 当前主要学习 Kotlin + Jetpack Compose。
- 实习中会使用 AI 辅助开发，但希望自己掌握关键决策和底层逻辑。
- 第一个项目偏通知栏/无障碍自动记账方向，已经接触过一些应用层架构。
- 第二个项目希望日常可用、可现场展示、技术深度更强。
- 用户晚上通常有约 2 小时学习和开发时间。

## 实施注意事项

- 初期骨架必须保持可编译。
- 真实传感器和 AudioRecord 实现应隐藏在 DataSource / Repository 后面。
- 权限请求应非阻塞，先解释用途，再引导授权。
- 不应在 App 启动时连续请求多个权限。
- 演示数据必须明确标注，不能伪装成真实采样数据。
- 核心模块要帮助用户理解，而不是只生成代码。
