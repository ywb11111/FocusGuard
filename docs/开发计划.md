# FocusGuard 开发计划

## 目标

根据 `docs/FocusGuard项目方案.md` 搭建并逐步完善 FocusGuard：Compose App 壳、导航、分层架构、领域模型、Repository 边界、Room/DataStore/Hilt/WorkManager、专注计时、传感器、音频、前台服务和报告能力。

## 上下文

- 用户是软件工程大三学生，方向为 Android，技术路线为 Kotlin + Jetpack Compose。
- 项目要兼顾日常可用和面试展示价值，重点覆盖音频、传感器、前台服务、本地持久化、Flow、权限和性能优化。
- 当前仓库已经不是空项目，基础骨架和内存版专注计时已完成。

## 决策记录

- 项目以最终的 FocusGuard 方案为准，不走早期 BLE-first 方向。
- 保留 domain 层，便于讲清业务模型、规则分析和工程边界。
- 初期允许使用演示数据，但必须明确它不是真实传感器或真实数据库数据。
- 核心学习模块优先让用户理解或亲自动手，AI 不直接全包。

## 已遇到的问题

| 问题 | 尝试 | 解决 |
|---|---|---|
| GitHub curated skill 列表返回 HTTP 403 | 运行 `list-skills.py --format json` | 改为读取本机 skill 目录 |
| 早期 `read_thread` 参数被工具拒绝 | 使用可选参数读取旧线程 | 改为补全参数后成功读取 |
| Gradle wrapper 在普通沙箱超时 | 运行 `:app:assembleDebug` | 使用批准后的提权构建 |
| AGP 9 内置 Kotlin 与 kapt/Hilt 组合冲突 | 尝试正常 Hilt+kapt | 切到 AGP 8.13.2 + Kotlin Android plugin + KSP |
| AndroidX 检查失败 | 缺少 `android.useAndroidX` | 在 `gradle.properties` 添加 `android.useAndroidX=true` |
| Kotlin/Javac target 不一致 | Kotlin 使用 JVM 21，Java 使用 11 | 设置 Kotlin `jvmTarget = JVM_11` |
| `kotlinOptions` 弃用警告 | 使用旧 DSL | 迁移到 `compilerOptions` |
