# FocusGuard Agent 协作规则

## 语言偏好

- 与用户对话默认使用中文。
- 代码注释、说明文字、项目文档、总结、计划、错误分析等内容，除非用户特别要求，默认使用中文。
- 保留代码标识符、API 名称、库名、命令、路径等必要英文，不为了中文化而破坏工程可读性。

## 项目协作偏好

- 做项目开发前，优先阅读 `AI实施规范.md`、`Git提交规范.md`、`docs/FocusGuard项目方案.md`、`docs/开发计划.md`、`docs/项目背景与发现.md`、`docs/开发进度.md`。
- 做项目时，如果环境中已安装或可用 `superpower` skill，应优先启动/使用该 skill，再继续开发。
- 若 `superpower` skill 不存在，应明确说明未安装或不可用，不要假装已经使用。
- 每完成一个小功能或一个可验证闭环后，必须提醒用户进行一次 Git 提交。
- 项目有结构、流程、阶段、规范或关键代码更新时，及时同步更新相关文档和必要注释。

## 教学偏好

- 用户希望在 AI 辅助开发中真正学会 Android 工程能力。
- 实现关键模块时，需要解释数据流、架构边界、关键代码和面试讲法。
- 不要只给代码；涉及 ViewModel、StateFlow、Room、SensorManager、AudioRecord、Foreground Service、权限、性能优化时，要帮助用户理解。

## 动手边界

- 建议用户亲自动手实现：核心学习模块的第一版，例如 ViewModel 状态流、Room DAO 基础查询、SensorManager 回调转 Flow、AudioRecord RMS 计算、前台服务通知创建。
- AI 可以生成代码但用户负责理解：页面样式、重复模板、Room 样板、Hilt 绑定、文档整理、测试样板、图表样式。
- AI 生成核心代码时，要标出用户应重点阅读的位置和需要能复述的逻辑。
